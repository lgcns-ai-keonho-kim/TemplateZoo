# API 모듈 개요

현재 API 계층은 단일 Agent 실행, 상태 확인, 정적 UI 경로를 제공한다.

## 공개 엔드포인트

| 영역 | Method | Path | 상태코드 | 설명 |
| --- | --- | --- | --- | --- |
| Agent | `POST` | `/agent` | `200` | 단일 요청 실행 |
| Health | `GET` | `/health` | `200` | 서버 생존 확인 |
| UI | `GET` | `/ui` | `200` | 정적 단일 요청 화면 |

## 설계 원칙

1. `POST /agent`는 요청 1건을 실행하고 결과를 단일 응답으로 반환한다.
2. `GET /health`는 서버 상태를 확인하고 `GET /ui`는 수동 검증용 정적 화면을 제공한다.
3. 라우터는 `AgentService` 단일 실행 경계만 주입받는다.
